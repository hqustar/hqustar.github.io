hexo主题样式文件修改无效果

2017年11月14日
| 阅读

hexo换了个themes 部署上去显示一切正常。想给header添加一个背景色，在浏览器测试成功。

然后修改themes下的样式文件 header.styl 增加一行代码

hexo clean hexo d -g 发现网页header 没有变化，查看网页源码 header 样式中没有

background #000

认识stylus

为什么是styl后缀文件，这个和传统的css样式文件有什么区别？ stylus介绍 官方Stylus API stylus中文版参考文档之综述

查看生成在public文件夹下的静态网页

.header {
  width: 100%;
  height: 110px;
  display: flex;
  align-items: center;
  flex-direction: row;
  flex-wrap: nowrap;
}
background #000 .table {
  display: table;
  height: 100%;
  width: 100%;
}
background #000 .table .connect {
  display: table-cell;
  vertical-align: middle;
}
background #000 .table .connect .connect-inner {
  box-sizing: border-box;
  padding-top: 8px;
  display: flex;
  flex-direction: column;
}

发现background #000 没有归到header 块里面，而是凸出来了。难不成是代码缩进出现问题？！！

问题出在讨厌的空格和TAB缩进

检查 header.styl 发现真得是代码缩进导致的错误 将TAB也改成四个空格后，header背景色就出现了。
